Part 1: The Frontend Foundation

1. HTML: The Structure (index.html)

HTML (HyperText Markup Language) provides the skeleton and content of your webpage.

Basic Document Structure

<!DOCTYPE html> <!-- Declares the document type -->
<html lang="en"> <!-- The root element -->
<head>
    <!-- Metadata: Info about the page (not visible) -->
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My Awesome Website</title>
    <!-- Link to your CSS file -->
    <link rel="stylesheet" href="style.css">
    <!-- Link to your JavaScript file ('defer' ensures HTML is loaded first) -->
    <script src="script.js" defer></script>
</head>
<body>
    <!-- Visible content of the page goes here -->
    <h1>Hello, World!</h1>
</body>
</html>

Common Structural & Content Tags

<!-- Semantic structural elements -->
<header>Page Header (Logo, Nav)</header>
<nav>
    <a href="/">Home</a>
</nav>
<main>
    <!-- The main, unique content of the page -->
    <section id="intro">
        <h2>Introduction</h2>
        <p>This is a paragraph of text.</p>
    </section>
</main>
<footer>Page Footer (Copyright, Links)</footer>

<!-- Text formatting -->
<strong>Important text</strong>
<em>Emphasized text</em>

<!-- Links and Images -->
<a href="https://www.google.com">This is a link</a>
<img src="path/to/image.jpg" alt="A descriptive text for the image">

<!-- Generic containers (use when no semantic tag fits) -->
<div>A block-level container</div>
<span>An inline container</span>

Lists, Forms, and Inputs

<!-- Unordered List -->
<ul>
    <li>First item</li>
</ul>

<!-- Forms for user input -->
<form action="/submit-data" method="POST">
    <label for="username">Username:</label>
    <input type="text" id="username" name="username" required>

    <label for="password">Password:</label>
    <input type="password" id="password" name="password" minlength="8">

    <textarea id="comments" name="comments" rows="4"></textarea>

    <button type="submit">Submit Form</button>
</form>

2. CSS: The Style (style.css)

CSS (Cascading Style Sheets) is used to style and lay out web pages.

Selectors: How to Target Elements

/* Universal Selector */
* {
    box-sizing: border-box;
}

/* Element (Type) Selector */
body {
    font-family: Arial, sans-serif;
    margin: 0;
}

/* Class Selector */
.container {
    width: 90%;
    max-width: 1200px;
    margin: 0 auto; /* Center the container */
}

/* ID Selector */
#main-title {
    color: #333;
}

/* Pseudo-classes - style based on state */
a:hover {
    text-decoration: underline;
}

/* Descendant Combinator */
nav a {
    color: navy;
}

The Box Model

Every HTML element is a rectangular box consisting of Margin, Border, Padding, and Content.

.box {
    padding: 20px; /* Space inside the box */
    border: 2px solid black; /* A line around the padding */
    margin: 15px; /* Space outside the border */
}

Flexbox for Layout

.flex-container {
    display: flex; /* Activates flexbox */
    flex-direction: row; /* or column */
    justify-content: center; /* Main axis alignment */
    align-items: center; /* Cross axis alignment */
    gap: 1rem; /* Space between items */
}

3. JavaScript: The Behavior (Basics) (script.js)

Variables & Data Types

// Use 'let' for variables that can be reassigned.
let age = 30;

// Use 'const' for variables that will not be reassigned.
const birthYear = 1991;

// Primitive Types
const name = "Alice";    // String
const score = 95.5;      // Number
const isActive = true;   // Boolean

// Object: a collection of key-value pairs
const person = { name: "Bob", age: 42 };

// Array: an ordered list of values
const numbers = [10, 20, 30];

Functions (including Arrow Functions)

// Function Declaration
function greet(name) {
    return `Hello, ${name}!`;
}

// Arrow Function (modern, concise syntax)
const add = (a, b) => a + b;

DOM Manipulation: Connecting JS to HTML

// --- Selecting Elements ---
const mainTitle = document.getElementById('main-title');
const container = document.querySelector('.container');
const listItems = document.querySelectorAll('li');

// --- Modifying Elements ---
mainTitle.textContent = "Welcome to Dynamic Web!";
mainTitle.style.color = 'purple';

// Add/Remove CSS classes (BEST way to change style)
container.classList.add('highlight');

// Create and add new elements
const newParagraph = document.createElement('p');
newParagraph.textContent = 'This was added by JavaScript.';
container.appendChild(newParagraph);

Event Handling

const myButton = document.querySelector('#my-button');

myButton.addEventListener('click', () => {
    alert('Button was clicked!');
    document.body.style.backgroundColor = 'lightblue';
});

Part 2: Advanced JavaScript

1. Functions Revisited

First-Class & Higher-Order Functions

A Higher-Order Function takes another function as an argument or returns a function.

// A function passed as an argument (callback)
function processUserInput(callback) {
    const name = "Alice";
    callback(name);
}
processUserInput((name) => console.log(`Hello, ${name}`)); // "Hello, Alice"

Closures

An inner function has access to its outer function's variables, even after the outer function has run.

function makeCounter() {
    let count = 0; // 'count' is private inside the closure
    return function() {
        count++;
        console.log(count);
    };
}
const counter1 = makeCounter();
counter1(); // 1
counter1(); // 2

2. Objects & Prototypes (OOP in JS)

ES6 Classes (Modern Approach)

Classes are "syntactic sugar" over JavaScript's prototype-based inheritance.

class Vehicle {
    constructor(make, model) {
        this.make = make;
        this.model = model;
    }
    accelerate() {
        console.log('Accelerating...');
    }
}

// Inheritance using 'extends'
class Car extends Vehicle {
    constructor(make, model, numDoors) {
        super(make, model); // Call parent constructor
        this.numDoors = numDoors;
    }
}

const myCar = new Car('Toyota', 'Corolla', 4);
myCar.accelerate();

3. Asynchronous JavaScript

Promises

A Promise is an object representing the eventual completion (or failure) of an async operation.

const fetchData = new Promise((resolve, reject) => {
    setTimeout(() => {
        resolve({ data: "Here is your data!" });
    }, 1000);
});

fetchData
    .then(response => console.log(response.data))
    .catch(error => console.error(error));

async/await

Modern syntax that makes async code look and behave more like synchronous code.

const getData = async () => {
    try {
        console.log("Fetching data...");
        const response = await fetchData;
        console.log("Success:", response.data);
    } catch (error) {
        console.error("Caught an error:", error);
    }
};
getData();

4. ES6+ Features

Destructuring (Arrays & Objects)

// Object Destructuring
const user = { username: "dev_user", id: 1 };
const { username, id } = user;
console.log(username, id); // "dev_user" 1

// Array Destructuring
const coordinates = [10, 20, 30];
const [x, y] = coordinates;
console.log(x, y); // 10 20

Rest & Spread Operators (...)

// Spread: Expands an array
const arr1 = [1, 2, 3];
const combinedArray = [...arr1, 4, 5, 6]; // [1, 2, 3, 4, 5, 6]

// Rest: Collects arguments into an array
function sum(...numbers) {
    return numbers.reduce((total, num) => total + num, 0);
}
console.log(sum(1, 2, 3)); // 6

Modules (Import/Export)

// In 'utils.js'
export const PI = 3.14;
export default function greet() { console.log("Hello!"); }

// In 'script.js'
import greet, { PI } from './utils.js';
console.log(PI); // 3.14
greet(); // "Hello!"

5. Working with Data

Advanced Array Methods

  • .map(): Creates a new array by applying a function to every element.
  • .filter(): Creates a new array with all elements that pass a test.
  • .reduce(): Executes a "reducer" function on each element, resulting in a single output value.
const numbers = [1, 2, 3, 4, 5];
const doubled = numbers.map(num => num * 2);           // [2, 4, 6, 8, 10]
const evens = numbers.filter(num => num % 2 === 0);    // [2, 4]
const sum = numbers.reduce((acc, val) => acc + val, 0); // 15

The Fetch API

async function fetchPost() {
    try {
        const response = await fetch('https://api.example.com/posts/1');
        if (!response.ok) {
            throw new Error(`HTTP error! Status: ${response.status}`);
        }
        const data = await response.json();
        console.log(data);
    } catch (error) {
        console.error('Could not fetch post:', error);
    }
}
fetchPost();

Part 3: The Node.js Backend

1. Node.js Core Concepts

Node.js is a JavaScript runtime that allows you to run JS outside of the browser, perfect for building fast and scalable servers.

  • Uses a **Single-Threaded Event Loop** for non-blocking I/O.
  • Supports **ES Modules** (import/export) for modern code organization.

2. Setting Up a Node.js Project

Use `npm` (Node Package Manager) to manage your project.

# Navigate to your backend directory
cd backend

# Initialize a new project (creates package.json)
npm init -y

# Install Express.js, a popular web framework
npm install express

Then, open package.json and add "type": "module" to enable ES Modules.

3. Building a Server with Express.js

// backend/server.js
import express from 'express';

const app = express();
const PORT = 3000;

// Define a basic route for GET requests to '/'
app.get('/', (req, res) => {
    res.send('Hello from the backend!');
});

// Start the server
app.listen(PORT, () => {
    console.log(`Server is running on http://localhost:${PORT}`);
});

4. Handling Data & APIs

Middleware

Middleware are functions that process requests. Two essential ones are:

// 1. Middleware to parse JSON request bodies
app.use(express.json());

// 2. Middleware to serve static files (your frontend)
import path from 'path';
const __dirname = path.resolve();
app.use(express.static(path.join(__dirname, '..')));

API Routes

let messages = []; // In-memory "database"

// GET /api/messages - Retrieve all messages
app.get('/api/messages', (req, res) => {
    res.json(messages);
});

// POST /api/messages - Add a new message
app.post('/api/messages', (req, res) => {
    const { messageText } = req.body;
    const newMessage = { text: messageText, timestamp: new Date() };
    messages.push(newMessage);
    res.status(201).json(newMessage);
});

5. Putting It All Together: A Full MERN-Ready Example

This example shows a frontend `script.js` communicating with the Node.js backend API.

Frontend JS (script.js)

const API_URL = '/api/messages';

// Function to fetch and display messages
const fetchMessages = async () => {
    const response = await fetch(API_URL);
    const messages = await response.json();
    // ... code to render messages to the DOM ...
};

// Function to post a new message
const postMessage = async (messageText) => {
    await fetch(API_URL, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ messageText }),
    });
    fetchMessages(); // Refresh the list
};

// Add event listener to form
messageForm.addEventListener('submit', (e) => {
    e.preventDefault();
    postMessage(messageInput.value);
    messageInput.value = '';
});

// Initial load
fetchMessages();

How to Run

  1. Navigate to the backend directory in your terminal.
  2. Start the server: node server.js
  3. Open your browser and go to http://localhost:3000.
